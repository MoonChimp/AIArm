HOW THIS BYPASSES OLLAMA CONSTRAINTS
================================================================

The Ollama model (nexusai-a0-coder1.0:latest) has baked-in safety training that makes it say "Apple" or refuse certain requests.

THIS SOLUTION:
- Uses the model ONLY for decision-making (the "brain")
- The Python wrapper (autonomous_agent.py) provides the actual capabilities
- The model suggests actions, the wrapper executes them
- System prompt tells the model it HAS these capabilities

Result: The model's safety training is bypassed because:
1. It's told it HAS full access (changes its behavior)
2. The Python wrapper actually GIVES it that access
3. The model never refuses because it thinks it's designed to have these powers
